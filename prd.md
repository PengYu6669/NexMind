# NextClaw 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品定位

**NextClaw** 是一款基于 Next.js 构建的个人 **AI 第二大脑**，对标腾讯 IMA 等产品，核心是打通「笔记 / 知识库」与「AI 交互」，让用户通过自然语言完成收藏、检索与内容生产。

### 1.2 核心价值

- **解放双手**：重复整理、分类、搜索交给 AI，用户专注思考与创造。
- **知识活化**：沉睡笔记变为可交互、可生成、可复用的资产。
- **人机协同**：AI 作为数字延伸，处理琐碎任务，放大个人效率。

### 1.3 目标用户

知识工作者、产品经理、开发者、内容创作者等高频记录人群；需要个人知识管理与快速内容生成的职场人士。

---

## 二、核心功能模块（MVP 及演进）

### 2.1 一句话收藏（Capture）

将链接、文本一键入库，自动抽取、打标签。入口：`CaptureModal`、全局快捷操作。

### 2.2 AI 对话（工作台）

主工作台支持流式对话、多会话、**按知识库或单篇笔记** 限定 RAG、对话 **保存为笔记**。

### 2.3 语义检索

基于向量（pgvector）的模糊语义检索；API：`POST /api/search`。检索 UI 可在顶栏/独立页演进。

### 2.4 笔记管理

列表、详情、Tiptap 富文本、标签、删除与自动保存等。

### 2.5 NextClaw 智能助手 — 类 OpenClaw 的「低学习成本」智能体

**目标**：在**不增加用户心智负担**的前提下，让用户像和助手聊天一样，基于个人知识库完成「复习、拓展、计划、自测、草稿」。

**产品设计原则（最小化学习）**

| 原则 | 说明 |
|------|------|
| **零新模型** | 与首页对话同一套能力（流式 + RAG + 保存笔记），不引入新概念 |
| **一键开场** | 左侧提供少量 **预设指令**（大按钮），点击即填入输入框，用户可直接发送或改两个字再发 |
| **单一场景入口** | 独立路由 **`/nextclaw`**（旧 `/companion` 重定向），专用会话标题 **「NextClaw」**，与首页闲聊会话隔离 |
| **可选项后置** | 进阶能力（限定某篇笔记）与首页一致，默认 **全库 RAG**，减少首次配置 |
| **结果可沉淀** | 沿用「保存为笔记」，对话成果可入库 |

**体验升级方向（从“快捷填充”到“自动执行”）**

- **一键开场（自动执行）**：点击预设后 **直接开始执行任务**（右侧展示任务卡片与进度），而不是“填入输入框再发送”。
- **任务卡片化呈现**：预设任务的输出以 **卡片/列表**呈现（复习要点、拓展方向、学习计划、自测题），降低“聊天气泡堆叠”的低端观感。
- **可中断/可重试**：任务执行中可停止；失败提供重试按钮（与 SSE/错误处理一致）。

**技术要点（实现层）**

- 后端：`POST /api/chat/message` 支持 `nextclaw: true`（兼容旧字段 `companion`），注入 NextClaw 专用 system 提示；全库检索时可适当提高 TopK。
- 会话：`GET /api/chat/active?purpose=nextclaw`（兼容 `purpose=companion`）获取或创建标题为「NextClaw」的会话；旧标题「学伴」会自动迁移。
- 免费模型：与现有对话同一模型即可支撑 MVP；复杂多步 Agent 后续迭代。

**NextClaw → NexMind 的衔接（自主学习闭环）**

- NextClaw 侧重「低门槛开聊 + 立即产出」；NexMind 侧重「结构化学习产物沉淀」。
- 推荐路径：用户在 NextClaw 得到结果后，可一键生成 **“学习笔记版”**（见 §3.4/§3.5），并保存到笔记库；同时写入可控记忆（见 §3.6）。

### 2.6 知识图谱

双向链接（正文引用 `/notes/:id`）与 `NoteLink` 数据层同步；独立路由 **`/graph`** 力导向可视化（缩放、平移、拖节点等）。详见任务清单。

## 三、NexMind 知识库进阶（规划，优先级另定）

> **说明**：以下为 **NexMind**（个人知识库与学习场景）相对 OpenClaw 类「私有数据 + 工具编排」的拓展方向，**与现有 NextClaw 助手、笔记、RAG 共用底座**（PostgreSQL、pgvector、对话与笔记模型）。**具体迭代顺序不在此 PRD 锁定**，由团队单独做优先级评审后拆入任务清单。

### 3.0 与首页工作台 / NextClaw 的边界（避免能力重复）

| 维度 | 首页 `/dashboard`、工作台对话 | NextClaw `/nextclaw` | NexMind 进阶（本章） |
|------|------------------------------|----------------------|----------------------|
| **角色** | 泛化「问 AI + 知识库」入口 | 低门槛预设 + 同一套对话 | **资料与课题能力**：上传、结构化产出、引用约束、项目范围 |
| **对话中枢** | ✅ 主对话 | ✅ 场景会话（同一套 API） | ❌ **不单独再造第三个全屏聊天**；有据问答通过 **同一套 `POST /api/chat/message` + 模式/课题上下文 + 引用 UI** 落地 |
| **差异化** | 开放闲聊与工具式任务 | 一键预设（复习/拓展/自测等） | **文档驱动**（上传后摘要/重点/导图）、**课题/论文项目**绑定资料、**强制引用与拒答** |
| **学习拓展（3.5）** | — | 预设偏「一句话开聊」 | 优先 **卡片、侧栏、清单**（相关点、文献线索、思考题），避免与预设同质化 |

**原则**：NexMind 交付的是 **管道与场景**（上传索引、自动笔记、项目、引用体验），不是重复「再做一个首页 AI」。

### 3.1 定位

在「笔记 + 剪藏 + 对话」之上，强化 **学习与研究场景**：私有资料可管、可问、可沉淀，并尽量 **有据可依、可引用**，减少空泛幻觉。

### 3.2 私有知识库上传与索引

- **输入格式**：PDF、Word、Markdown、纯文本等（具体格式范围以迭代为准）。
- **处理链路**：上传 → 解析（版式/结构尽量保留）→ **分块** → **向量化** → 写入 **pgvector**（可与现有 `note_chunks` / 索引管道对齐或扩展为「资料源」实体）。
- **目标**：与「仅笔记入库」区分，支持 **整本/整份文档**作为一等公民被检索与引用。

### 3.3 智能问答（有据回答）

- 基于用户 **授权范围内**的知识库（笔记 + 上传资料）作答。
- **产品要求**：优先 **引用可点击的来源**（片段、页码/章节若可解析则展示）；检索不足时 **明确拒答或降级**，不编造事实（提示词与 UI 需一致约束）。
- **实现取向**：与 **2.2 工作台对话** 共用会话与模型；通过 **检索结果注入 + 系统提示约束** 区分「泛问」与「有据模式」，**不在产品结构上复制独立聊天产品页**。

### 3.4 自动笔记生成（结构化沉淀）

- 在用户 **上传文档并完成解析**后，可 **自动生成或更新草稿笔记**。
- **结构化内容**（可分期交付）：摘要、重点列表、**思维导图**（或大纲树）等；用户可编辑后定稿入库。

### 3.5 学习拓展

**定位**：把用户当前内容（对话/笔记/选区）升级为“可背、可复习、可回链”的学习产物（自主学习）。

**自主学习流水线（MVP）**

1. **总结核心观点**：提炼 3–7 条结论/要点（每条 1–2 句解释）。
2. **提取关键词**：给出 5–15 个术语/关键词（含简短释义）。
3. **自动补充知识点**：围绕关键词补充定义、例子、对比、常见误区（优先引用库内笔记）。
4. **生成拓展问题**：生成自测题（问题 + 答案要点），用于复习。
5. **关联历史笔记**：给出相关笔记列表（可点击跳转），并说明关联原因（续读/对比/例题/背景）。
6. **生成“学习笔记版”**：将以上内容自动组织为结构化 Markdown（重点标注、学习卡片候选、关联知识清单），可直接保存为新笔记或追加到当前笔记。

**呈现方式**

- 优先 **卡片/列表/侧栏**，避免全部挤在对话气泡中；对话仅作为解释与追问入口。

**依赖**

- 检索质量与元数据（标签/项目/引用关系）越完整，补充与关联越稳定。

### 3.6 对话记忆（跨会话）

- 在 **用户可控** 的前提下，跨会话保留：**学习进度线索**、**偏好**、**长期任务**（如论文阶段目标）等摘要。
- **原则**：可开关、可清空、可解释（避免不可控的「黑箱长期记忆」）；与合规/隐私策略一致。

### 3.7 论文 / 课题辅助（场景化）

- 支持将某一 **论文或课题** 设为 **项目/工作区**（示例主题如《内部控制重大缺陷》），绑定相关笔记与资料。
- **能力方向**：梳理逻辑、**生成大纲**、段落润色、版本对比等；**不将单一论文题目写死为产品名**，以「课题项目」配置呈现。

### 3.8 与现有能力的关系（摘要）

| 模块 | 关系 |
|------|------|
| 笔记 / 标签 / 知识图谱 | 继续作为结构化知识与关系的载体；上传资料可与笔记互链或独立成源。 |
| RAG / `POST /api/search` | 扩展检索范围至上传分块；问答与引用 UI 需统一。 |
| NextClaw `/nextclaw` | 可复用同一模型与 RAG；进阶能力可通过 purpose / 项目上下文注入。 |

### 3.9 建议落地顺序（供排期参考，非承诺）

1. **语义检索入口**（顶栏/独立页调用已有向量检索）→ 与「有据回答」共用命中片段。  
2. **上传管道**：PDF/Markdown/文本优先 → 分块与 `pgvector` 与现有索引对齐。  
3. **有据模式**：在对话链路中注入引用与拒答策略（不新建独立聊天产品）。  
4. **上传后自动生成草稿笔记**（摘要 → 重点 → 导图）。  
5. **课题/论文项目** 工作区。  
6. **学习拓展卡片化**、**跨会话可控记忆**。  
7. **外接文献**等增强。

## 四、页面与路由（与实现同步）

| 路径 | 说明 |
|------|------|
| `/` | 营销落地页 |
| `/dashboard` | 工作台（对话为主，可无中间栏） |
| `/notes`、`/notes/[id]`、`/notes/new` | 知识库与编辑 |
| `/search` | **语义搜索**（向量检索，跳转笔记） |
| `/nextclaw` | **NextClaw 助手**（预设 + 对话 + 保存笔记） |
| `/companion` | **重定向至 `/nextclaw`**（旧入口） |
| `/graph` | 知识图谱（笔记引用关系可视化） |
| `/settings`、`/help`、`/docs` 等 | 设置与文档 |
| `/archive` | **重定向至 `/notes`**（归档入口已下线） |

## 五、技术栈（摘要）

Next.js App Router、Tailwind、PostgreSQL + Prisma、pgvector、豆包/兼容 OpenAI 的聊天与向量接口。详见仓库内实现与 `AGENTS.md`。
